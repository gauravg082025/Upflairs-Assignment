\pard\sa200\sl276\slmult1\f0\fs22\lang9 Advanced Static To-Do List (No State)\par
\f1\u-10180?\u-8273?\f2  \f0 Objective:\par
Create a static to-do list application using React that groups tasks by category, conditionally renders completion status, and displays detailed descriptions \f2\emdash  without using any state (useState, useReducer, etc.).\par
\par
\f3\u-10178?\u-8727?\f2  \f0 What You'll Practice:\par
JSX rendering\par
Array .map() and .filter()\par
Conditional rendering\par
Grouping data dynamically\par
Component structure and styling\par
\par
\f1\u-10179?\u-9013?\f2  \f0 Requirements:\par
1. Data Structure:\par
Inside your component, create a hardcoded array of todo items. Each item should contain:\par
\{\par
  id: 1,\par
  title: "Finish report",\par
  description: "Monthly financial report",\par
  category: "Work",\par
  completed: true\par
\}\par
Include at least 6 items, spanning 3 different categories (e.g., Work, Personal, Hobby).\par
2. Grouping by Category:\par
Dynamically extract unique categories from the data.\par
Render each category as a section with a heading.\par
List the corresponding tasks under each category.\par
3. Rendering Tasks:\par
For each task:\par
Show \f4\u9989?\f2  \f0 before the title if it's completed.\par
Show \f4\u10060?\f2  \f0 if it's not completed.\par
Below the title, render the description in a smaller or italic font.\par
4. No Interactivity:\par
This is a static assignment:\par
Do NOT use state hooks like useState.\par
Do NOT use event handlers like onClick.\par
\par
